# 预测模型设计

## 当前核心思路

系统不直接使用“去年排位 ≈ 今年排位”。

而是：

```text
历史趋势
+
排位百分位
+
考生规模归一化
+
保守趋势修正
```

---

# 第一步：排位百分位归一化

例如：

```text
2024 物理类最低排位 = 50000
2024 物理类总人数 = 450961
```

则：

```text
50000 / 450961 = 11.08%
```

转换为同科类排位占比。

这样可以解决：

```text
不同年份考生人数变化
```

导致的排位失真问题。

---

# 第二步：加权平均

当前使用：

```text
2024 = 0.78
2023 = 0.14
2022 = 0.06
2021 = 0.02
```

理由：

- 最近一年参考价值最大
- 旧年份只作为长期稳定参考
- 模型整体偏保守

---

# 第三步：残差趋势修正

趋势权重：

```text
0.80
0.15
0.05
```

趋势系数：

```text
0.20
```

目的：

- 识别专业升温/降温
- 避免纯平均数过于迟钝
- 但不让趋势影响过大

系统整体偏保守。

---

# 第四步：预测 2025 排位

```text
预测百分位 × 2025 对应科类人数
```

例如：

```text
11.08% × 440208
```

得到：

```text
2025 预测排位
```

---

# 冲稳保规则

当前界限：

| 类型 | 与预测排位差值 |
| --- | --- |
| 保 | <= -5% |
| 稳 | -5% ~ 5% |
| 冲 | 5% ~ 15% |
| 险 | > 15% |

---
